Automated YouTube Shorts from Tech News — End to End

Fetch real tech news → summarise with Hugging Face AI → voiceover → Pexels stock video → upload as a YouTube Short. Zero manual steps.

✍ Shweta Suryavanshi 📅 April 24, 2026 ⏱ 8 min read

// TL;DR — key takeaways

  • Phase 1 — Fetch & Store: Pull live tech news from free APIs, summarise each article with a Hugging Face model, and persist to a JSON file (slug = datetime) with an unprocessed flag.
  • Phase 2 — Produce & Upload: For each unprocessed news item, convert the summary to speech (TTS), overlay the audio onto a random tech video from the Pexels API, and upload the result as a YouTube Short.
  • Both phases run independently on Vercel cron schedules — fetch/summarise runs first, upload runs after — so the pipeline is resilient to partial failures.
  • All credentials (News API key, HF token, Pexels API key, YouTube OAuth) live in environment variables; nothing sensitive is committed to the repo.

The Idea: Turn Tech News into Daily Shorts, Automatically

Tech moves fast. A lot of people want quick video summaries of what happened today in the industry — but producing even a 45-second video manually takes 20–30 minutes when you factor in scripting, recording, editing, and uploading. I wanted to close that gap entirely.

The pipeline is split into two decoupled phases. Phase 1 is a data pipeline: it fetches news, summarises it with AI, and stores it locally as a JSON "queue". Phase 2 is a media pipeline: it drains that queue, producing and uploading one YouTube Short per news item. Keeping them separate means a failed upload never blocks new news from being fetched and summarised.

Phase 1 — Fetch, Summarise & Store

The first cron job runs every few hours. It calls two free tech-news APIs — NewsAPI and GNews — filtering for the technology category and deduplicating by article URL before doing anything else. Fresh articles are then batched and sent to Hugging Face's facebook/bart-large-cnn summarisation model to produce a 2–3 sentence plain-English summary — short enough for a 45-second voiceover. 

# src/news_fetcher.py
import requests, json, os
from datetime import datetime

NEWS_APIS = [
    f"https://newsapi.org/v2/top-headlines?category=technology&pageSize=10&apiKey={os.getenv('NEWS_API_KEY')}",
    f"https://gnews.io/api/v4/top-headlines?category=technology&token={os.getenv('GNEWS_API_KEY')}",
]

def fetch_articles():
    articles = []
    seen_urls = set()
    for url in NEWS_APIS:
        for item in requests.get(url).json().get("articles", []):
            if item["url"] not in seen_urls:
                seen_urls.add(item["url"])
                articles.append(item)
    return articles

Each article is summarised via the Hugging Face Inference API. The resulting records are appended to a JSON file named after the current datetime (e.g. 2026-04-24T10-00-00.json) and every record gets a "status": "unprocessed" field. This file is the handoff point between Phase 1 and Phase 2. 

# src/summariser.py — HF Inference API
import requests, os

HF_API = "https://api-inference.huggingface.co/models/facebook/bart-large-cnn"
HEADERS = {"Authorization": f"Bearer {os.getenv('HF_TOKEN')}"}

def summarise(text: str) -> str:
    payload = {"inputs": text, "parameters": {"max_length": 80, "min_length": 30}}
    resp = requests.post(HF_API, headers=HEADERS, json=payload)
    return resp.json()[0]["summary_text"]
# src/store.py — write slug-named JSON queue file
from datetime import datetime
import json, pathlib

def save_queue(articles: list):
    slug = datetime.utcnow().strftime("%Y-%m-%dT%H-%M-%S")
    path = pathlib.Path(f"data/{slug}.json")
    path.parent.mkdir(exist_ok=True)
    records = [
        {"title": a["title"], "url": a["url"],
         "summary": a["summary"], "status": "unprocessed"}
        for a in articles
    ]
    path.write_text(json.dumps(records, indent=2))
    return path

Phase 2 — Produce & Upload the Short

The second cron job runs shortly after Phase 1. It scans all JSON files in data/ for records with "status": "unprocessed", picks the oldest one, and runs it through a three-step media pipeline: TTS → Pexels video → YouTube upload.

Step A — Text-to-Speech: The summary text is converted to a .mp3 using gTTS (free, fast) or ElevenLabs (higher quality). The audio duration determines how long the final video will be — the stock clip is trimmed to match. 

# src/tts.py
from gtts import gTTS
import tempfile

def synthesise(text: str) -> str:
    tts = gTTS(text=text, lang="en", slow=False)
    tmp = tempfile.NamedTemporaryFile(suffix=".mp3", delete=False)
    tts.save(tmp.name)
    return tmp.name   # path to audio file

Step B — Fetch a Pexels Stock Video: The Pexels Videos API is queried with the keyword "technology" (randomised page offset so the same clip isn't reused every day). The first portrait-orientation clip (aspect ratio ≥ 0.5) is downloaded — this ensures the video is already close to the 9:16 Shorts format. 

# src/pexels_client.py
import requests, os, random, tempfile

PEXELS_API = "https://api.pexels.com/videos/search"
HEADERS = {"Authorization": os.getenv("PEXELS_API_KEY")}

def fetch_video() -> str:
    page = random.randint(1, 20)
    resp = requests.get(PEXELS_API,
        headers=HEADERS,
        params={"query": "technology", "orientation": "portrait",
                "per_page": 5, "page": page})
    video_url = resp.json()["videos"][0]["video_files"][0]["link"]
    tmp = tempfile.NamedTemporaryFile(suffix=".mp4", delete=False)
    tmp.write(requests.get(video_url).content)
    return tmp.name

Step C — Composite & Upload: moviepy trims the stock clip to the audio duration, overlays the news headline as an animated text caption, and mixes in the TTS audio track. The output is a 1080×1920 .mp4. The YouTube Data API v3 then uploads it as a Short with an auto-generated title, description, and hashtags derived from the news headline. After a successful upload the record's status is flipped to "processed" in the JSON file. 

# src/video_builder.py (simplified)
from moviepy.editor import VideoFileClip, AudioFileClip, TextClip, CompositeVideoClip

def build_short(video_path, audio_path, headline) -> str:
    audio = AudioFileClip(audio_path)
    clip  = VideoFileClip(video_path).subclip(0, audio.duration)
    caption = (TextClip(headline, fontsize=48, color="white", font="Arial-Bold",
                        method="caption", size=(clip.w - 80, None))
               .set_position(("center", 0.75), relative=True)
               .set_duration(audio.duration))
    final = CompositeVideoClip([clip, caption]).set_audio(audio)
    out   = video_path.replace(".mp4", "_short.mp4")
    final.write_videofile(out, fps=30, codec="libx264",
                          ffmpeg_params=["-vf", "scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:(ow-iw)/2:(oh-ih)/2"])
    return out

Cron Scheduling on Vercel

Both phases are deployed as Vercel Serverless Functions. Phase 1 (fetch + summarise) runs at 0 */4 * * * — every 4 hours — to keep the queue topped up. Phase 2 (produce + upload) runs at 30 */4 * * * — 30 minutes after Phase 1 — giving it time to finish before the upload job starts. 

// vercel.json
{
  "crons": [
    { "path": "/api/fetch-news",    "schedule": "0 */4 * * *" },
    { "path": "/api/produce-short", "schedule": "30 */4 * * *" }
  ]
}

All secrets — NEWS_API_KEY, GNEWS_API_KEY, HF_TOKEN, PEXELS_API_KEY, and the YouTube OAuth refresh token — are stored as Vercel environment variables. The data/ JSON files are committed back to the repo via the GitHub Contents API after each Phase 1 run, so Phase 2 always has access to the latest queue even in a stateless serverless environment.

What's Next

Right now the caption is just the raw headline overlaid as static text. The next step is word-by-word caption animation — synced to the TTS audio timestamps — which makes Shorts significantly more watchable and improves retention. I'd also like to add a thumbnail generator: take the headline, render it on a branded background with Pillow, and set it as the YouTube video thumbnail via the API.

Longer term, the same two-phase architecture maps onto Instagram Reels and TikTok with minimal changes: swap the upload module, adjust the output resolution slightly, and add a second entry to vercel.json. The news fetching, AI summarisation, TTS, and Pexels video steps stay identical.
